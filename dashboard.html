Developer Productivity Agent | Python · LangGraph · LlamaIndex · ChromaDB · MCP · FastAPI · Groq

▸ Built a LangGraph-orchestrated autonomous developer agent using a plan-and-execute
  loop with automatic replanning: a planning node decomposes each query into 2–5
  [tool_name]-prefixed sub-tasks; an execution node calls the correct tool directly
  by parsing the prefix (avoiding LLM tool-calling API limitations); a replan node
  revises remaining steps on failure; and a synthesis node assembles a structured
  final answer — all persisted to ChromaDB session memory for future retrieval.

▸ Implemented four registered tools: LlamaIndex semantic search over a ChromaDB
  vector index of project documentation (BGE-small-en-v1.5 embeddings, 512-token
  chunks), DuckDuckGo live web search with retry/backoff, a sandboxed Python
  subprocess executor (10s timeout, dangerous import blocklist), and a ChromaDB
  session memory retriever that embeds past debugging sessions for semantic lookup.

▸ Exposed all four tools as an MCP-compatible server using FastMCP, served the
  agent via FastAPI POST /query, and evaluated across 50 annotated developer queries
  spanning documentation lookup, code generation, debugging, and multi-step workflows
